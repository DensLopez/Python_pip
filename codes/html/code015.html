Qu es docker?

Docker es una herramienta que tambin nos sirve para aislar entornos.

Recapitulemos de cmo hemos aislado nuestros proyectos

  1. Primero tenamos una versin compartida de Python y mdulos que se poda enlazar o comunicar entre s.

  1. Luego, con los entornos virtuales de Python logramos aislar cada dependencia y cada modulo a ca proyecto pero conservando o teniendo en comn a Python como una versin global, con esto logramos que no exista un choque entre proyectos y sus dependencias. Este tipo de ambiente es muy comn en Ciencia de Datos y algunas otras areas.

  1. Cuando ya necesitamos llevar el cdigo a produccin y servidores en la nube nos encontramos con un problema, y es que tambin necesitamos aislar la versin de Python. Porque tambin nos podemos encontrar con proyectos en los cuales las versiones de Python son diferentes. Por ejemplo podemos tener un proyecto con Python 2.7, otro con Python 3.8 o Python 3.10. Entonces lo que hace Docker es aislar no solo las dependencias, sino tambin el entorno de ejecucin y con ello las versiones que estamos corriendo de Python.

Esto lo hace con contenedores, una tecnologa que posee una gran caracterstica que es; as como funciona la aplicacin en mi computadora, se puede realizar un despliegue para que funcione en cualquier equipo; ya sea en la nube (Amazon. Google Cloud) o local.

Un ejemplo que podramos desplegar es un web server en la nube. En las prximas clases veremos como funciona este tipo de tecnologa.

Aporte de Juan Camilo Rubio Rodriguez

Aqu tienes 5 razones por las que Docker es esencial:

  • Portabilidad y Consistencia: Docker encapsula una aplicacin junto con todas sus dependencias, bibliotecas, y configuraciones en un contenedor. Esto garantiza que la aplicacin funcionar de la misma manera en cualquier entorno, ya sea en un equipo de desarrollo, un servidor de pruebas o en produccin, eliminando problemas de "funciona en mi mquina".
  • Aislamiento de Aplicaciones: Docker permite ejecutar mltiples aplicaciones en contenedores separados en la misma mquina, sin que interfieran entre s. Cada contenedor tiene su propio entorno, lo que evita conflictos entre versiones de software o dependencias.
  • Despliegue Rpido y Escalabilidad: Docker permite desplegar aplicaciones rpidamente y escalar horizontalmente sin complicaciones. Puedes lanzar nuevas instancias de una aplicacin en contenedores en cuestin de segundos, lo que facilita la respuesta a cambios en la demanda.
  • Eficiencia de Recursos: Los contenedores de Docker son ligeros y utilizan menos recursos que las mquinas virtuales tradicionales porque comparten el mismo sistema operativo base. Esto permite que ms contenedores se ejecuten en la misma infraestructura, reduciendo los costos de hardware y mejorando la eficiencia.
  • Integracin con CI/CD: Docker se integra perfectamente con flujos de trabajo de integracin continua y despliegue continuo (CI/CD). Permite automatizar la construccin, prueba y despliegue de aplicaciones de manera confiable y reproducible, acelerando el ciclo de desarrollo y mejorando la calidad del software.
